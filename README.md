# Aspectos claves de Git y Git Hub

1. ¿Qué es Git?

Es un software de control de versiones. Es de código abierto, gratuito, robusto y descentralizado. Básicamente es nuestro repositorio local.

2. ¿Qué es Git Hub?

Es una plataforma que permite alojar proyectos utilizando el sistema de versiones de Git. Es una interfaz gráfica en la web de nuestro repositorio remoto.

3. ¿Qué es el staging area?

Área donde se almacenan de manera temporal los archivos preparados que estarán en el commit.

4. ¿Qué es un commit?

Es una instantánea de nuestro proyecto en cuanto a los cambios realizados.

5. ¿Qué comando utilizo para agregar archivos modificados al Staging Area?

```
git add <nombre-del-archivo-modificado>
```
6. ¿Qué comando utilizo para agregar archivos preparados al repositorio?

```
git commit -m "mensaje"
```
El mensaje debe ser descriptivo para tener claridad sobre la información que aloja.

7. ¿Cuál es el nombre por defecto de la rama principal de mi proyecto?

Puede ser main o master y lo definimos al momento de la instalación de Git.
